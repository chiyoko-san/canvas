# 🧩 絵文字当て・今日の一問

毎日0時に全世界共通の絵文字クイズが1問出る、Wordle型のデイリーゲーム。
「🦁👑 = ?」のような絵文字を見て答えを当てる。連勝記録とシェアが軸。

## ゲームルール
- **毎日0時に1問**、全世界で同じお題（日付ベースで全員同一）
- 絵文字を見て答えを**文字で入力**。3回まで挑戦できる
- **表記ゆれ吸収**：カタカナ⇄ひらがな、全角⇄半角、記号/空白/長音/中黒を無視、部分一致も許容
- ヒントは1回だけ開放できる
- **連勝記録**：毎日正解すると伸びる。1日でも外す/逃すと0に
- **正解した日数・正解率**を記録
- **全世界の正解率**をFirebaseで集計表示（「全世界の正解率72%」）
- **Wordle風シェア**：🟩⬜🟥で結果を表現、答えはネタバレせずに自慢できる

## 仕組み
- フロント：静的HTML 1枚（`index.html`）→ GitHub Pages
- バックエンド：Firebase Realtime Database（全世界の正解率の集計だけに使用）
- 連勝・統計はlocalStorage管理
- お題は内蔵の30問を日替わりループ（`PUZZLES` 配列）

### DB構造
```
emojiquiz/day{N}
  ├ solved   その日の正解者数
  └ total    その日の挑戦者数
```

## セットアップ

### 1. firebaseConfig を貼る
`index.html` の該当箇所に貼り替え。THE BUTTON等と同じプロジェクトを使い回せる
（ツリーが `emojiquiz` で別なので衝突しない）。

### 2. データベースのルール
Realtime Database →「ルール」タブに `database.rules.json` の中身を反映。
既存のツリーに `emojiquiz` を足した統合版になっている。

### 3. GitHub Pages
新規リポジトリ（例 emoji-quiz）を作って push → Settings → Pages で `main` / `(root)`。
公開URL：`https://＜ユーザー名＞.github.io/emoji-quiz/`

---

## お題の追加・編集
`index.html` の `PUZZLES` 配列を編集。形式：
```js
["絵文字", ["答え", "表記ゆれ1", "表記ゆれ2"], "ジャンル", "ヒント"]
```
答えは配列で複数登録すると表記ゆれに強くなる（例：["ライオンキング","らいおんきんぐ","thelionking"]）。
問題数を増やすほど、日替わりループが一巡するまでの日数が延びる。

## カスタム
- `MAX_TRIES` … 挑戦回数（既定3）
- `dayNumber()` の起点日 … いつを Day 0 にするか

## 拡張アイデア
- 全世界の連勝ランキング
- ジャンル選択（映画だけ・ことわざだけ等）
- 友達と同じ問題で競う対戦リンク
- 答えた人の分布（何回目で正解したか）

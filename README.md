# 🎨 世界をひとつの絵にする

全世界の人が1マスずつ塗って、ひとつの絵を完成させる共有ドット絵キャンバス。
r/place の小型版。50×50＝2500マス、16色、1マス塗るごとに30秒のクールダウン。
譲り合いと陣取りが自然に生まれる協力ゲーム。

## 仕様
- **50×50キャンバス**を全世界で共有、リアルタイム反映
- タップで選択中の色を塗る。**30秒に1マス**のクールダウン
- **16色パレット**（r/place相当）
- ピンチで拡大・2本指/拡大中1本指でパン（細かいマスもスマホで塗れる）
- 塗られたマス数・自分の貢献数を表示
- 白ベースの見やすいデザイン

## 仕組み
- フロント：静的HTML 1枚（`index.html`）→ GitHub Pages
- バックエンド：Firebase Realtime Database（無料枠）
- 同期：`canvas/pixels/{マス番号} = 色番号(0-15)` の差分更新。塗った1マスだけを書き込むので軽い
- ※クールダウンはlocalStorage管理（ゆる制限。データ消去で突破可能）

### DB構造
```
canvas
  ├ pixels/{0..2499}   各マスの色番号（0-15）
  └ totalPlaced        累計設置数
```

## セットアップ

### 1. firebaseConfig を貼る
`index.html` の該当箇所に貼り替え。THE BUTTON / 投票所と同じプロジェクトを使い回せる
（ツリーが `canvas` で別なので衝突しない）。

### 2. データベースのルール
Realtime Database →「ルール」タブに `database.rules.json` の中身を反映。
既存の global/groups/meta/topics に `canvas` を足した統合版になっている。

### 3. GitHub Pages
新規リポジトリ（例 world-canvas）を作って push → Settings → Pages で `main` / `(root)`。
公開URL：`https://＜ユーザー名＞.github.io/world-canvas/`

---

## カスタム（index.html上部）
- `SIZE` … キャンバスの一辺（既定50）
- `COOLDOWN_MS` … クールダウン（既定30000=30秒）
- `PALETTE` … 16色の配列

## 拡張アイデア
- 塗った場所に名前が残る「誰が塗ったかログ」
- タイムラプス（履歴を保存して再生）
- 期間限定でキャンバスをリセットして毎回お題を変える
- 色ごとの使用数ランキング
